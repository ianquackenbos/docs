## How savings tracking works

<Steps>
<Step title="Day 0: Capture baseline">
When you apply a recommendation, Shardly captures:
- Current deployment costs
- Affected indices costs
- Storage sizes and shard counts
- Tier distribution

This baseline becomes the "before" snapshot for comparison.
</Step>

<Step title="Days 1-7: Monitor costs">
Shardly tracks costs daily during the measurement period:
- Collects cost data hourly
- Accounts for normal day-to-day variation
- Monitors for any regression or issues
- Tracks affected indices specifically

<Info>
The 7-day window accounts for weekday/weekend usage patterns and ensures accurate measurements.
</Info>
</Step>

<Step title="Day 7: Calculate savings">
After 7 days, Shardly calculates actual savings:

```
Baseline Cost  = Average daily cost (days -3 to 0)
Current Cost   = Average daily cost (days 4 to 7)
Daily Savings  = Baseline - Current
Monthly Savings = Daily Savings Ã— 30
```

<Check>
Savings are now verified and displayed in your dashboard
</Check>
</Step>

<Step title="Ongoing: Continued monitoring">
Shardly continues tracking for 30 days to:
- Detect cost regression
- Ensure savings persist
- Update confidence scores
- Alert on unexpected changes
</Step>
</Steps>

### Confidence score

Shardly assigns a confidence score (0-100%) to indicate measurement reliability:

<CardGroup cols={4}>
<Card title="90-100%" icon="check-double">
**Excellent**

High confidence, stable data, reliable savings
</Card>

<Card title="70-89%" icon="check">
**Good**

Good confidence, some variation, trustworthy
</Card>

<Card title="50-69%" icon="circle-half-stroke">
**Moderate**

Moderate confidence, noisy data, needs review
</Card>

<Card title="<50%" icon="circle-xmark">
**Low**

Low confidence, unstable, extend measurement
</Card>
</CardGroup>

#### Factors affecting confidence

<AccordionGroup>
<Accordion title="Data stability" icon="wave-square">
Lower cost variation = higher confidence. If costs fluctuate wildly day-to-day, confidence decreases.
</Accordion>

<Accordion title="Measurement period" icon="calendar">
Longer measurement periods increase confidence. 7 days is minimum; 30 days is optimal.
</Accordion>

<Accordion title="External factors" icon="cloud">
Traffic spikes, data growth, or other changes during measurement reduce confidence.
</Accordion>

<Accordion title="Implementation quality" icon="wrench">
Clean implementations without issues or rollbacks have higher confidence.
</Accordion>
</AccordionGroup>

## Best practices

<AccordionGroup>
<Accordion title="Wait for verification" icon="hourglass">
Don't implement multiple optimizations simultaneously. Wait 7 days between implementations for accurate savings attribution.
</Accordion>

<Accordion title="Share wins with stakeholders" icon="bullhorn">
Use the ROI report generator monthly to showcase value to leadership and finance teams.
</Accordion>

<Accordion title="Track by team" icon="users">
Use cost attribution to show each team their optimization impact and build optimization culture.
</Accordion>

<Accordion title="Monitor for regression" icon="bell">
Set up alerts for savings regression so you can address issues before they negate your optimizations.
</Accordion>

<Accordion title="Use confidence scores" icon="gauge-high">
Only report savings with >70% confidence to stakeholders. Lower confidence needs more data.
</Accordion>
</AccordionGroup>

## API access

Access savings data programmatically:

<CodeGroup>
```bash cURL
# Get all verified savings
curl -X GET "https://api.shardly.io/api/v1/savings" \
  -H "Authorization: Bearer $TOKEN"

# Get savings for specific recommendation
curl -X GET "https://api.shardly.io/api/v1/recommendations/rec-123/savings" \
  -H "Authorization: Bearer $TOKEN"
```

```python Python
import requests

response = requests.get(
    "https://api.shardly.io/api/v1/savings",
    headers={"Authorization": f"Bearer {api_token}"}
)

savings_data = response.json()
total_savings = sum(s["actual_monthly_savings"] for s in savings_data)
print(f"Total verified savings: ${total_savings:,.0f}/month")
```
</CodeGroup>
